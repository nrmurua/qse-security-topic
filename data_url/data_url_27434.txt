This issue was originally flagged in PR #125.
In summary, it isn't clear that a PulseSequence object should start at t=0. This is not explicit in the documentation, but it is an underlying assumption for a number of instruments. To rectify this, we should clarify the documentation, and perhaps assert that no Pulse.t_start can be less than 0, nor can a Pulse.duration be less than 0 (though I assume this is already the case.
